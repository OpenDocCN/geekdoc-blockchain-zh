# 概念列表

| 账本 | 一本账目、一份清单或其他财务账户和/或交易的集合。 |
| --- | --- |
| 51%攻击 | 对区块链网络的一种潜在攻击，在这种攻击中，一个单独的实体或组织能够控制大部分的哈希能力，可能会导致网络中断。在这种情况下，攻击者拥有足够的挖矿能力，可以故意排除或修改交易的顺序。 |
| 6-blocks 确认 | 添加一个包含一些交易的解决块后需要的时间（通常约为 60 分钟），以使交易得到确认。在此期间，创建一个不包含给定交易的新版本区块链变得过于困难，因为计算能力过于昂贵。 |
| 替代币 | 除了比特币之外的加密货币。 |
| 人工神经网络 | 一种基于连接单元或节点集合的计算系统，这些单元或节点被称为人工神经元，它们松散地模拟生物大脑中的神经元。 |
| 比特币 | 最受欢迎的加密货币，运行在一个由未知人士在 2009 年 1 月创建的区块链上，此人士使用化名中本聪（Satoshi Nakamoto）。 |
| 比特币/数字钱包 | 用于持有和交易比特币的软件程序，它使用个人的私钥（秘密数字）来访问个人的公共比特币地址（由 26-35 个字母数字字符组成的标识符）和交易签名，这些需要安全地存储；比特币钱包有多种形式——主要四种类型是桌面、移动、网络和硬件。 |
| 区块链 | 由一系列交易块组成的分布式账本，其中每个块（除了第一个和最后一个）通过加密手段与前一个和下一个块链接，从而形成一个不可破解的块链。 |
| 区块链分叉 | 在社区内（区块链 P2P 网络的节点/成员）发生的一种情况，当网络中存在两个或多个版本的区块链时（即，不同节点可能具有包含区块链数据的文件的不同的版本）。 |
| 区块链挖矿 | 一种点对点的计算机过程，旨在安全和验证给定加密货币（如比特币）的交易。 |
| 边界网关协议 | 路由 IP 数据包的协议（路由协议）；这是一种标准的 exterior 网关协议，旨在在互联网上的自治系统之间交换路由和可达性信息。 |
| 分支尖端 | 代表分支尖端包（参见 IOTA 包）中的最后一个单独交易的尖端。 |
| 拜占庭容错系统 | 一个能够抵抗拜占庭将军问题的系统；在严格意义上，一个 BFT 系统只要被破坏的元素数量不超过总元素数量的三分之一，就能正常运行。 |
| 拜占庭将军问题 | 一种系统故障类别，其中系统的某些组件/元素可能被损坏，以至于它们表现出阻止其他系统组件相互达成协议（即共识）的症状（外观、行为）。 |
| 车对车通信 | 车辆之间进行的通信，旨在共享可用于交通控制或警告驾驶员危险的信息。 |
| 共识机制 | 一种确保所有参与者都拥有分布式数据库文件（即分布式账本）相同副本（规则和协议的集合）。 |
| 协调器（在 Tangle 的上下文中） | 由 IOTA 基金会实施的一种机制，旨在保护 Tangle 免受漏洞的侵害，如双重支付（由于 Tangle 尚未完全发展，即 IOTA 网络中的节点总数和哈希能力相对较低）；协调器机制假定每两分钟发布一个里程碑交易。 |
| 加密货币 | 一种使用加密技术来规范货币单位生成和验证资金转移的数字货币，独立于中央权威机构运作，即不依赖于中央银行；大多数加密货币基于区块链技术。 |
| 累计权重 | 在 Tangle 中的随机游走算法中用于选择尖端的度量标准；在 Tangle 中，交易的累计权重是后来所有交易（加 1）的总数，同时直接或间接地批准了该交易。 |
| 去中心化应用 | 在去中心化计算系统上运行的计算机应用程序。去中心化应用由如以太坊区块链之类的分布式账本技术普及；智能合约是 DApps 的关键部分。 |
| 延迟攻击 | 针对比特币 P2P 网络的攻击，也利用 Border Gateway Protocol 的弱点（与分区攻击类似）；在延迟攻击中，块向受害节点的传播被有效减慢，即延迟；主要目标是让受害节点几乎 20 分钟内无法获得新块的信息，这是通过攻击者修改从受害节点到受害对等节点的消息实现的；由于攻击者修改了受害者的查询消息，受害者节点无法从网络获取最新的已解决块（以将其添加到其区块链副本中）；受害者浪费了其计算能力和时间，而不是用于解决下一个块；因此，受害者失去了可能的收益。 |
| 服务拒绝攻击 | 一种网络攻击，使连接到互联网的主机（计算机、机器、网络资源）不可用；通常通过向目标主机发送过量请求来实现，目的是过载系统并阻止合法请求的完成。 |
| 难度 | 比特币区块链中实现的算法，用于调节解决工作量证明（PoW）任务的难度（区块解决），使得平均每 10 分钟解决一个区块（即 14 天内解决 2016 个区块）；作为一个参数，它被定义为 Difficulty = Difficulty_1_target/Target_value，表示在当前目标值（由参数 Target_value 确定）下找到一个 256 位长的哈希值比在 256 位参考目标值（由参数 difficulty_1_target 确定，且有 32 个前导零）下找到一个哈希值困难多少倍。 |
| 难度目标 | 区块头部中的一个 32 位字段，包含参数 Target_value，通过 24 位尾数和 8 位指数表示；这个字段以十六进制表示的内容也被称为位数。 |
| 数字孪生 | 潜在和实际物理资产（物理孪生）、过程、人员、地点、系统和设备的数字副本，可用于各种目的。 |
| 有向无环图 | 一个没有有向周期的有限有向图。 |
| 分布式账本 | 复制、共享和同步的数字数据，在多个地理位置、国家或机构间进行共识。 |
| 双哈希（或哈希-哈希）算法 | 通常，双哈希是一种计算机编程技术，用于解决哈希冲突，当冲突发生时使用密钥的二次哈希作为偏移量；在比特币区块链中，双哈希通过双倍使用 SHA-256 哈希函数实现，即 f(x) = sha256(sha256(x))。 |
| 双花问题 | 数字现金方案中的一个潜在缺陷，其中同一个数字代币可以被花费多次；如果两个交易（其中一个很狡猾）使用欺诈性发送者的同一个私钥（在两个不同交易中的买家）发送到不同的接收地址（在两个不同交易中的卖家），就会发生这种情况；当两个交易在网络中传播时，网络的一部分将接受一个交易，另一部分接受另一个交易。 |
| Ethash | 以太坊网络和基于以太坊的加密货币实现的 PoW 挖矿算法；与 SHA-3 一样，Ethash 也是一种属于同一（Keccak）哈希函数家族的哈希函数，但与 SHA-3 不同；Ethash 的开发重点是保护 Application Specific Integrated Circuits 矿工，他们主要挖掘比特币。 |
| 以太币 | 以太坊区块链的原生代币（加密货币）；最初，以太币并没有被看作是一种常见的数字货币，其主要用途是作为在线支付商品和服务的方式，而是一种激励机制来推动以太坊区块链。 |
| 以太坊区块链 | 一个去中心化、开源的区块链，具有智能合约功能（一个于 2013 年开始的项目，由 Vitalik Buterin 领导）。它是市值第二大的加密货币平台（仅次于比特币）。 |
| 事件（在 hashgraph 的上下文中） | 节点在接收到其他（发送）节点的八卦/同步消息时刻在 hashgraph P2P 网络中创建的一个数据结构；它包含一个有效载荷（发送节点从之前接收到的八卦/同步消息中学到的关于新交易和其他节点间八卦的信息），然后是一个时间戳和两个指向称为自父事件和他父事件的前两个事件的哈希值。 |
| 著名见证人（在 hashgraph 的上下文中） | 在一轮中，被选为该轮的著名见证人的见证人，在两个（或更多）上级轮次的见证人进行的虚拟投票程序中；这是大多数节点在创建后相对很快了解到的一个见证人。 |
| 法定货币/法定货币 | 一种不由商品（如黄金）支撑的政府发行的货币。 |
| 创世事件/创世区块 | 在 Tangle/IOTA 中创建的第一个交易，其中创建了整个 IOTA 代币的总量。 |
| 八卦协议 | 基于疾病传播方式的计算机 P2P 通信的一种过程或方法；一些分布式系统使用它来确保数据传播到一组的所有成员；在一些临时网络中，八卦协议假设成员会将接收到的数据转发给他们的邻居。 |
| 保证（或安全）区块 | 在区块链中的一个区块，在此区块之后增加了六个或更多的新的区块，它被认为是区块链分叉不可能存在于保证区块之前的区块链区域。 |
| 哈希率/算力 | 计算设备每秒能执行的哈希计算次数。 |
| 哈希值 | 固定长度的一个数值，唯一标识（数据的一部分），是数据经过哈希函数处理的结果；由于比特币区块链使用了哈希算法，即对输入数据（交易区块的头部）进行两次 SHA-256 哈希函数处理，区块头部（实际区块的）的哈希值是 256 位长。 |
| Hashgraph | 由 Leemon Baird（于 2016 年）开发并由 Hedera Hashgraph LLC 支持的受许可的分布式账本/平台，该平台应用 hashgraph 算法——一种新的八卦协议（逻辑上基于 DAG），能够实现完整的拜占庭容错共识机制；除了原生加密货币（Hedera 代币）之外，该平台旨在提供其他服务，如文件存储和智能合约，即无需许可的去中心化应用程序的发展。 |
| Hedera Hashgraph 理事会 | 旨在支持基于 Hashgraph 共识算法的稳定和去中心化公共账本基础设施发展的 governing body。 |
| Hedera Hashgraph LLC | 启动 Hashgraph 项目的公司；为项目的支持提供公共账本平台。 |
| 隐藏尖端 | 由于 IOTA 网络中的传播延迟，节点尚未看到的尖端。 |
| 工业 4.0 | 用作第四工业革命的含义；因为它还没有完全定义，它更指的是具有一系列新兴技术和工具的集合，这些技术和工具有潜力大幅改进各种制造过程；它还指的是传统工业过程与数字化之间的协同作用，通过信息通信技术帮助行业中的机器和过程智能联网，从而执行制造解决方案。 |
| 输入交易（在交易捆绑中） | 包含负值“Value”字段的交易（在交易捆绑中），意味着字段“Address”指定的账户将按“Value”字段中的 IOTA 数量收费。 |
| 物联网（IoT） | 具有唯一标识符和能够在不需要人与人或人与计算机交互的情况下通过网络传输数据的相互连接的计算设备、机械和数字机器的系统。 |

| 挖矿池 | 一群联合的加密货币矿工，他们在网络上组合计算资源以解决 PoW 任务并将事务区块添加到区块链；如果池成功完成 PoW，它将用加密货币代币奖励；奖励根据每个矿工的处理能力或工作相对于整个小组的比例分配给贡献者。 |
| 多宿主 | 将主机或计算机网络连接到多个网络的做法。通常这样做是为了提高可靠性或性能。 |
| 节点 | 某个特定区块链上的 P2P 网络的参与者，运行该区块链的原生币/加密货币或去中心化应用。 |
| 随机数 | 区块链矿工解决的唯一一次使用的数字，即搜索，在将新区块添加到区块链之前；它是放在交易区块（比特币区块链）头部的一个 32 位值，满足难度级别限制，即块头（包含随机数）的 256 位哈希值足够小，即具有所需的前导二进制零；对于每个块，块头的哈希值（包含已解决的随机数）放在下一个块的头部，以此方式形成一个不可破解的交易区块（即区块链）链。 |
